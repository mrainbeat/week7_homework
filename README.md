# 👾 7주차 과제 - Delivery (배달 주문 웹 애플리케이션)

피그마 시안의 디자인을 반영하고, 로컬스토리지(`localStorage`)를 활용하여 상태 변화를 실시간으로 추적합니다. 사용자가 로그인 후 원하는 음식점을 클릭하면 상세 메뉴와 정보를 확인하고 주문 후 결제창까지 넘어가는 리액트 기반 웹 애플리케이션입니다.

---

## 🚀 주요 기능

### 1. 로그인 / 로그아웃 / 회원가입
* 사용자 인증 상태에 따라 메뉴 페이지 및 장바구니 접근 권한을 제어합니다.
* 해당 계정에서 선택한 가게와 음식을 로컬 스토리지에 저장하여 장바구니에서 실시간으로 확인할 수 있습니다.

### 2. 네비게이션 바 
* 반응형으로 구현하여 sh일때는 햄버거 바가 생성됩니다
* 장바구니 알림 아이콘 위에 현재 담은 메뉴의 개수가 렌더링됩니다.

### 3. 카테고리별 필터링
* 카테고리 버튼을 클릭하면 해당하는 음식점 목록만 화면에 실시간으로 렌더링합니다.

### 4. 음식점 카드 UI
* Tailwind CSS를 활용하였으며, 화면 해상도 변화에도 깨지지 않는 반응형 그리드 레이아웃 시스템을 구축했습니다.

### 5. 상세 정보 모달창
* 특정 음식점 카드를 클릭하면 세부 메뉴가 있는 모달 창이 렌더링됩니다.
* 모달 내부에는 클릭한 음식점의 이름, 별점, 그리고 목데이터(Mock Data)로부터 매핑된 세부 메뉴 리스트가 출력됩니다.

### 6. 장바구니 및 결제 
* 메뉴 페이지에서 선택했던 음식을 장바구니 페이지에서 바로 확인할 수 있습니다.
* 결제 수단 선택 및 총 가격 확인 기능을 지원합니다.

## 📐 레이아웃 및 디자인 가이드

피그마 시안에 명시된 단위를 레이아웃에 정밀하게 반영하였으며, 유연한 CSS 기법을 사용했습니다.

* **로그인 / 회원가입 페이지**
  * **입력 칸(Input):** 모바일 환경에서 부모 상자를 벗어나지 않도록 너비 리셋 및 `box-sizing: border-box` 설정을 적용했습니다.
  * **인터페이스:** 독립적인 버튼 및 입력 칸 형태로 깔끔하게 구현했습니다.
* **장바구니 및 결제 페이지 (Order)**
  * **구조:** 왼쪽 장바구니 상자는 배경색과 그림자 효과를 제거하여 전체 배경과 동화되는 형태로 구성하고, 오른쪽 결제 카드는 흰색 카드 레이아웃을 유지합니다.
  * **화면 배치:** 브라우저 세로 공간을 동적으로 계산하여 네비게이션 바 하단 구역에서 상하좌우 정중앙에 위치하도록 구현했습니다.

## 💾 핵심 로직 및 데이터 연동

### 메뉴 컴포넌트 및 로컬스토리지 연동
메뉴 페이지와 결제 페이지 간의 데이터 연동은 브라우저 공용 저장소인 `localStorage`를 매개체로 작동합니다.
